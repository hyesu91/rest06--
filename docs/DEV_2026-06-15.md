# rest06 개발 로그 — 2026-06-15

## 개요
`dev/rest06.html`(Claude 디자인 아티팩트, 22MB 단일 번들)을 기반으로
드림아이티비즈 IT 교육 플랫폼을 **React + Vite + Supabase** 스택으로 구축.

## 한 일

### 1. 아티팩트 분해 & 자산 추출
- 번들에서 뷰 템플릿(112KB) + 컨트롤러 JSX(26KB) 추출.
- 임베드 폰트(JetBrains Mono / Space Grotesk / Pretendard, 22MB) → CDN 링크로 대체.
- 원본은 `reference/rest06.original.html` 로 보존(요청대로 전체 업로드).

### 2. DC 템플릿 엔진 재구현 (`src/dcRuntime.jsx`)
원본 dc-runtime 문법을 React 렌더러로 1:1 포팅:
- `<sc-for list as>` 반복, `<sc-if value>` 조건, `{{ expr }}` 보간, `$index` 루프 인덱스
- `onclick="{{ }}"` → onClick, `style-hover` → 마우스 호버, `style` → React style 객체

### 3. 앱 컨트롤러 (`src/App.jsx`)
- 아티팩트의 `Component`(DCLogic) → `React.Component` 포팅: 테마(6색)·다크/라이트·
  화면 라우팅(home/online/cert/ai/community/detail/enroll/dashboard 등)·카운터/로테이터 fx.
- React 리렌더가 덮는 테마/모드 CSS 변수는 `componentDidUpdate` 에서 항상 복원.

### 4. 백엔드 연결
- **인증**(`lib/auth.js`): Supabase OAuth 구글·카카오 로그인, 세션 구독, 프로필 upsert.
  네비 "로그인" 버튼 → 로그인/계정 오버레이 모달(React, 디자인 외부 레이어).
- **결제**(`lib/payment.js`): 아임포트(PortOne v2) SDK 지연로드 + 결제요청, 결제·수강 기록.
  키 미설정 시 테스트 모드로 흐름 검증. 무료체험은 결제 없이 수강신청.
- 템플릿 최소 패치: "수강 신청하기"→결제화면, "결제하고 시작하기"→`doPay`, 신청자 정보→로그인 유저.

### 5. 인프라
- `vite.config.js` base `/`, `public/CNAME`=rest06.dreamitbiz.com.
- `.github/workflows/deploy.yml`: main push → 빌드(시크릿 주입) → Pages 배포.
- `sql/schema.sql`: `rest06_profiles / rest06_payments / rest06_enrollments` + RLS.

## 검증
- `npm run build` 성공 (JS 271KB / gzip 74KB).
- jsdom + renderToStaticMarkup 로 DC 런타임·전체 App 렌더 무오류 확인
  (DREAM·무료체험·결제·로그인 출력 확인). 모든 `{{ }}` 바인딩이 renderVals 로 충족됨.

## 남은 연동 (대시보드에서 키/설정만)
- Supabase: schema 실행 + Auth Providers(Google/Kakao) 활성화 + Redirect URL 등록.
- GitHub Secrets: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`,
  `VITE_PORTONE_STORE_ID`, `VITE_PORTONE_CHANNEL_KEY`.
- PortOne: 결제대행사 채널 연결.
- GitHub Pages: Settings > Pages > Source = **GitHub Actions** (build_type 전환 확인).
